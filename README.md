# 2027年度エンジニアインターンシップ：事前学習課題

本資料は、開発未経験（または経験が浅い）学生が、インターンシップ当日のチーム開発にスムーズに合流できるよう、最低限必要な知識とスキルを習得するためのガイドです。以下の課題レベルについては最低限自分で一度は取り組んだことがあるレベルを想定します。開発未経験の方は必ず一通り目を通してください。

**目標学習時間：約24時間（8時間×3日間）**

---

## 📅 DAY 1：開発の土台を作る（Git / GitHub / VS Code）

**目標：コードを保存し、インターネットを通じて共有・公開できる。**

### 1-1. 環境構築

1. **Visual Studio Code (VS Code)** をインストールする。
2. **Git** をインストールする。
3. **GitHub** アカウントを作成する。

### 1-2. 課題：GitHubリポジトリの提出

1. GitHub上に `eiger-internship-pre-study` という名前のリポジトリを作成する（Public設定）。
2. ローカル（自分のPC）でフォルダを作成し、Gitで初期化する。
3. `README.md` を作成し、そこに「インターンへの意気込み」を記載してコミットする。
4. GitHubへプッシュし、ブラウザで自分の意気込みが表示されていることを確認する。

### 1-3. 課題：ブランチとプルリクエストの体験

1. `feature/add-profile` という新しいブランチを作成する。
2. `profile.md` を追加し、氏名や好きな技術（または興味のあること）を記載してコミットする。
3. GitHubへプッシュし、「Pull Request」を作成する。
4. 自分で自分のプルリクエストをマージし、`main` ブランチに反映されることを確認する。

---

## 📅 DAY 2：環境をコンテナで揃える（Docker / SQL）

**目標：どのPCでも同じ開発環境を動かせる仕組み（Docker）と、データの扱い方を学ぶ。**

### 2-1. 環境構築

1. **Docker Desktop** をインストールする（起動することを確認）。
2. **DBeaver**（またはVS CodeのSQL拡張機能）をインストールする。

### 2-2. 課題：データベースを立ち上げる

1. 以下の内容で `docker-compose.yml` を作成する。
   ```yaml
   services:
     db:
       image: postgres:15
       environment:
         POSTGRES_USER: user
         POSTGRES_PASSWORD: password
         POSTGRES_DB: intern_db
       ports:
         - "5432:5432"
   ```
2. `docker compose up -d` コマンドでデータベースを起動する。
3. DBeaverから `localhost:5432` に接続し、データベースの中身が見れることを確認する。

### 2-3. 課題：SQLの基本操作

以下のSQLコマンドを実行し、その実行ログ（またはスクリーンショット）を保存する。

1. `users` テーブルを作成する（id, name, email）。
2. 自分のデータを1件 `INSERT` する。
3. `SELECT` 文でデータが保存されたことを確認する。

---

## 📅 DAY 3：フルスタックTODOアプリを自作する（実践）

**目標：ブラウザから操作できる、データベース連携型のTODOアプリを完成させる。**

これまでの学習を統合し、「画面（フロントエンド）」と「機能（バックエンド）」が連携する仕組みを構築します。

### 3-1. 開発の構成と課題

以下の2つのパーツを作成し、それらを連携させてTODOアプリを完成させてください。

#### ① バックエンド（APIの作成）

Python（FastAPI等）または Node.js を使い、データベースとやり取りする「窓口（API）」を作ります。

- **GET /todos**: DBからタスク一覧を取得し、JSON形式で返す。
- **POST /todos**: フロントエンドから送られてきたタイトルをDBに保存する。
- **PATCH /todos/{id}**: 特定のタスクの完了状態（`is_done`）を切り替える。

#### ② フロントエンド（UIの作成：HTML / CSS / TypeScript）

ブラウザで表示される「見た目」と「動作」を作ります。

- **HTML/CSS**: タスクの入力欄、追加ボタン、リスト表示エリアを作成し、使いやすくデザインする。
- **TypeScript**:
  - `fetch` 関数を使い、バックエンドAPIからデータを取得したり、新しいタスクを送信したりする。
  - APIから返ってきたデータを元に、`document.createElement` 等を使って画面を動的に書き換える。

### 3-2. データベースの構成（再確認）

TODOアプリ用に以下のテーブルを使用します（DAY 2 で作成または追加）。

- テーブル名：`todos`
  - `id`: シリアル（自動採番 / Primary Key）
  - `title`: テキスト（タスク内容）
  - `is_done`: ブール値（デフォルトは false）
  - `created_at`: タイムスタンプ（デフォルトで現在時刻）

### 3-3. 最終提出物

1. DAY 1 で作成した GitHub リポジトリに、以下のファイルを整理してプッシュしてください。
   - `backend/`: APIのソースコード
   - `frontend/`: HTML, CSS, TypeScriptのファイル
   - `docker-compose.yml`
2. **README.md**:
   - アプリの起動手順（例：Docker起動後、バックエンドを起動し、ブラウザでindex.htmlを開く、など）。
   - 画面のスクリーンショット。
   - 工夫した点（UIのデザイン、バリデーション、エラー時の処理など）。
   - 3日間の学習を通じて一番難しかったところ。

---

## 💡 学習のヒント

- **「疎通」が最優先**: 最初から凝ったデザインにせず、まずは「ボタンを押したらAPIが呼ばれる」「画面に文字が出る」という最小限の連携（疎通）を最優先しましょう。
- **CORSエラーへの対策**: ブラウザからAPIを叩く際、CORS制限でエラーが出ることがあります。「FastAPI CORS許可」「Express CORS設定」などで調べて解決策を実装してみてください。
- **デバッグのコツ**: プログラムが動かない時は、SQLを直接DBeaver等で実行してみて、DB側の問題かプログラム側の問題か切り分けましょう。
- **AIの活用**: 「TypeScriptでAPIから取得したデータをリスト表示するコード例を教えて」など、具体的なパーツごとにAIを活用するとスムーズです。
- **完璧主義を捨てる**: 24時間は短いです。まずは「動く」ことを最優先にしましょう。
